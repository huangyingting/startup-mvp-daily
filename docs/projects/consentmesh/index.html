为什么值得做

这件事现在值得做，是因为生成式 AI 的问题已经从“能不能生成”转向“生成所依赖的权利关系是否可持续”。第一，新闻明确指出当前市场仍主要靠事后投诉和下架来处理未授权使用，说明供需双方之间缺的不是更多法务口号，而是一套可执行的前置许可机制。第二，Loti 把受影响对象从明星和大品牌扩展到普通个体，说明可被 AI 使用的对象正在迅速大众化，传统一单一谈的授权方式根本撑不起规模。第三，文章同时点名 AI 开发者面临法律不确定性，意味着买方痛点同样强烈：平台需要可放心接入的、带许可边界的数据与身份资产。窗口期的本质，不在于又多了一类内容资产，而在于“许可条款”本身第一次需要被产品化、API 化、持续追踪化。

ConsentMesh 的创意，不是把“肖像权”或“版权”继续当成合同附件管理，而是把每一个人像、声音、作品都变成一个自带权限、用途边界、价格和分账规则的策略对象。这样产品的核心就不再是存证，而是执行：当某个 AI 平台要调用某段声音、训练某类模型或生成某种风格内容时，系统能像支付网关判断是否扣款那样，实时判断是否允许、记录证据、触发收益分配，并在超出边界时拦截或升级复核。

Problem

要解决什么问题？

今天无论是创作者、艺人经纪公司、品牌还是 AI 平台，都在一个非常原始的权利工作流里运转。创作者很难提前声明“我的声音可以用于哪些场景、不能用于哪些场景、训练和生成是不是两回事、商业广告和教育用途要不要分开计价”；平台也很难稳定拿到这些权限，并证明某次使用确实在许可范围内。于是市场只能退回最笨的办法：先让内容流动，再等侵权、投诉、下架、赔偿和公关危机发生。更麻烦的是，授权不只是一个 yes/no 判断，还涉及地域、期限、模型类型、可否二次训练、可否转授权、是否要求署名、收益如何分成等复杂条件。市场缺的不是更多“保护创作者”的口号，而是一层把许可、使用、归因、付款和撤回串成同一条操作链的基础设施。

目标用户

  • 管理艺人、KOL、主播、体育人物或品牌代言人数字权利的经纪公司、IP 授权团队与品牌保护负责人
  • 正在做 AI 视频、虚拟人、语音克隆、创意生成或模型训练产品，需要可解释许可来源与合规证据的 AI 平台产品团队与法务/合规团队
  • 希望把 UGC、创作者合作或 AI 广告素材生产做成可规模化业务，但不想继续靠邮件和 PDF 合同逐单对接的品牌营销团队与创作者经济平台

MVP

这个原型包含什么

如果一个轻量但可信的 rights exchange 能让个人和机构把肖像、声音与作品配置成机器可读的许可对象，并通过 API 直接接入 AI 平台的训练与生成流程，自动记录使用事件、生成归因证据并完成分账，那么创作者侧会把它当作新的授权入口，平台侧会把它当作新的合规与供给入口。因为它替代的是今天最脆弱、最慢、最容易引发争议的那段“先上车、再补法务”的 AI 权利协作流程。

权利许可编排器

把声音、肖像、图片、视频片段与创作素材配置成机器可读的许可对象，明确训练 / 生成、商业 / 非商业、可否二次分发、地域、期限、署名要求与撤回条件，让权利边界第一次可以被系统直接调用。

用途追踪账本

接收来自参与 AI 平台的训练请求、生成调用与内容发布事件，把每次使用映射到具体资产、具体许可版本和具体合作方，形成可审计的 usage ledger，而不是只留一堆截图和法务邮件。

归因与分账引擎

基于使用事件自动计算应付金额、分润比例与结算周期，把创作者、经纪方、品牌方和平台之间的收益分配标准化，减少每次都要重谈“到底怎么算钱”的交易摩擦。

撤回与争议闸门

当权利人修改授权范围、发现超范围使用或希望临时冻结某类场景时，系统能立即暂停相应调用、标记高风险生成结果并推送人工复核，避免争议只能在内容扩散后再补救。

Use case

创作者把声音与肖像授权给 AI 视频工具

很多合作卡住，不是因为创作者不愿意授权，而是没人能用统一方式讲清楚“能怎么用、用了多少次、钱怎么算”。

  • 创作者在 ConsentMesh 中上传自己的声音样本与形象素材，选择一套许可模板：允许用于品牌短视频生成，但不允许用于政治内容、成人内容和模型再训练。
  • 接入平台的 AI 视频工具在创建数字分身前先查询许可对象，只有符合边界的请求才能继续完成资产绑定和内容生成。
  • 每次生成与发布都会回写到用途账本，月底自动生成使用明细、归因报表和分账结果，创作者不用再靠人工对账判断自己是否被超范围使用。
北极星指标：从资产接入到首次被授权使用的周期 / 每位创作者的月度可追踪使用次数 / 超范围使用拦截率

Use case

经纪公司管理代言人数字分身 campaign

真正难的不是拿到一次授权，而是把一次授权稳定执行成几十个渠道、几百条素材都不出边界。

  • 经纪团队为某位艺人建立 campaign policy，限定可投放国家、授权日期、品牌名单、可使用台词范围与是否允许生成静态海报或动态视频。
  • 品牌与制作方在调用模型时只能从白名单素材与白名单场景里组合内容，任何超出期限或超出渠道的请求都会被系统直接拦下。
  • 活动结束后，经纪团队导出一份完整证据包，里面包含所有生成资产、实际投放记录、分账情况与异常拦截记录，方便复盘和续约谈判。
北极星指标：campaign 审批周转时间 / 未授权素材流出率 / 复盘证据包生成时间

Use case

AI 平台把侵权投诉前移为供应侧准入

平台最怕的不是有人投诉，而是根本不知道哪些内容本来就不该被系统用进去。

  • AI 平台把 ConsentMesh 接到训练数据导入和生成资产调用的关键路径，在资产进入系统前先校验许可状态与可用范围。
  • 当产品团队想上线新的语音克隆模板或广告生成场景时，系统自动列出可立即启用的合规资产池，以及缺少哪些授权条件才可放行。
  • 如果后续权利人撤回或修改边界，平台会收到影响范围清单，快速定位哪些模型作业、模板或客户工作流需要暂停或替换。
北极星指标：新场景上线前的法务审核工时 / 投诉触发率 / 因权利问题导致的生成任务回滚率

Stack

当前技术栈

  • 底层数据模型围绕 person/work asset、license scope、usage event、partner platform、attribution claim 与 payout rule 构建，把“内容文件”升级成“带权限的资产对象”。
  • 产品先从 Web 控制台 + partner API 起步，而不是试图做区块链式开放市场；早期重点是让经纪方、平台方和品牌方能先把核心合作流程跑通。
  • 许可引擎用规则系统处理地域、期限、用途、模型类型和敏感场景限制，保证关键边界可解释、可审计，而不是完全交给黑箱模型判断。
  • AI 主要用于相似内容识别、异常使用分类、素材去重与争议初筛，但最终的权利放行、撤回和高风险例外仍保留人工闸门。
  • 用途账本需要接入平台侧的生成、发布和训练遥测，并结合水印、指纹或素材签名能力，提高“实际用了什么”的可信度。
  • 分账层优先支持简单的 usage-based payout 和 campaign-based payout，两种模式先跑通，再扩展到更复杂的版税与多方收益拆分。

Risks

主要风险

  • 如果参与的 AI 平台太少，产品会退化成好看的授权面板，却无法真正验证使用和结算，网络效应是第一道门槛。
  • 肖像权、版权、声音权和 biometric data 的法律边界在不同司法辖区差异很大，跨境执行会显著增加产品复杂度。
  • 平台若不愿共享训练和生成遥测，usage ledger 容易退化成自报数据，削弱创作者对系统的信任。
  • 普通用户虽然也有权利需求，但若 onboarding 复杂、预期收入过低，早期更可能还是专业创作者和经纪机构先用起来。
  • 一旦系统把边界设错，错误授权或错误拦截都可能直接带来商业损失，因此早期必须把可解释规则和人工复核做重。

Next steps

如果继续迭代，接下来做什么

  • 先聚焦高价值、边界最清晰的场景，例如 AI 视频头像、语音克隆和品牌代言数字分身，把许可模板和 payout 规则做深。
  • 与 1-2 个真实生成平台建立双向事件回传，验证 usage ledger 是否足够可信，并把“接入后投诉下降多少”做成首批 ROI 指标。
  • 为经纪公司、品牌与平台分别提供标准 policy 模板，降低首次上线时的法务配置成本。
  • 第二阶段再把站外监测和维权联动接回来，形成“先授权可交易，后监测可追责”的完整 rights ops 闭环。
